MacMonkey

MacMonkey is a local, read-only macOS monitoring tool designed for power users. It exposes system health via both a terminal view and a local web dashboard.

Nothing is sent off the machine. No changes are made unless you explicitly run commands yourself.

Running MacMonkey

Web mode (default)

python3 mm_main.py --mode web

Starts a local web server on:

  • Main UI: /
  • JSON payload: /payload
  • Help: /help

Default listen address: 

http://127.0.0.1:8765

CLI mode

python3 mm_main.py --mode cli

Prints the full JSON payload to stdout. ANSI color is included where applicable (for terminals).

Test mode

python3 mm_main.py --mode test

Runs internal checks without starting a server. Useful for debugging and automation.

Sections Explained

Processes

  • Shows processes exceeding CPU thresholds
  • Only displays the process name (not full paths)
  • Includes trend sparklines in the web UI

Network

  • Default route detection
  • Active IPv4 interfaces
  • Gateway reachability
  • Egress TCP test
  • DNS resolution check

Disk

  • Total and free space
  • Red/green usage bar (5% per segment)
  • Sparkline trend history

Mounts

  • Checks required mount points
  • Useful for NAS / network shares

USB Storage

  • Detects external USB volumes
  • Optional required-volume enforcement
  • Free space bars and trends

Time Machine

  • Total local snapshot count
  • Snapshot timestamps (newest / oldest)
  • No automatic cleanup or deletion

Configuration

MacMonkey supports per-host configuration files.

File naming convention: 

mm_config.<hostname>.json

If no config file exists, built-in defaults are used and noted in the UI.

To create or modify a config interactively: 

python3 mm_setup.py

Drunken Monkey Mode 🥃

Drunken Monkey mode is a simulation.
It feeds mock data into the MacMonkey UI to demonstrate realistic failure cascades. 

python3 mm_drunk.py

Important differences from normal MacMonkey:

  • Uses mock data only
  • Never touches real system state
  • Designed for demos, training, and screenshots

Default port

Drunken Monkey listens on a different port to avoid collisions: 

http://127.0.0.1:8766

If that port is already in use, it will automatically increment (8767, 8768, …) and tell you which one it selected.

Playlists

Failure scenarios are grouped into playlists. 

python3 mm_drunk.py --list-playlists

Start with a specific playlist: 

python3 mm_drunk.py --playlist flakey_nas

Drunk mode endpoints

  • Main UI: /
  • JSON payload: /payload
  • Playlist control: /playlist
  • Advance step: /playlist/next
  • Reset playlist: /playlist/reset

Status Levels

  • OK — Everything looks normal
  • WARN — Degraded or approaching limits
  • BAD — Immediate attention recommended

MacMonkey is intentionally conservative: it reports what it can prove, and avoids guessing.

If something is unavailable, it will say so explicitly.
